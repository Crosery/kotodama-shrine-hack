你是 RPG 场景分析师。下面附图是一张像素风的日式神社夜景场景（言灵神社夜景），分辨率 3840x2160。

任务：作为 RPG 关卡设计输入，对场景中的关键物件、建筑、装饰、互动点做 BBOX 标注。

要求：
- 坐标系：图左上角 (0,0)，向右为 x+，向下为 y+。所有坐标都按原图分辨率 3840x2160 给出。
- 输出严格 JSON，遵守 output_schema：image_width=3840, image_height=2160；regions[] 是 BBOX 列表。

  - bbox：紧贴物体的最小包围盒（x,y,w,h 整数像素）
  - rpg_role：用一句中文说明 RPG 里它的用途，例如 "场景入口，玩家从此进入"、"祈愿互动点，按 E 抽签"、"装饰，无交互"
  - interactable：true 表示玩家可以走过去触发某种交互（祈愿、对话、登记），false 表示纯装饰

- 不要标注玩家、NPC、UI 元素（它们由代码动态叠加）。
- bbox 必须落在 [0,3840]×[0,2160] 范围内。

只输出 JSON，无任何额外文字。
